Configuration Tutorial

This tutorial explains the structure and options available in the custom web server configuration file.

Example Configuration

# example of a config file

server [
    port = 3000;
    servernames = localhost;
    bodysizelimit = 10MB;
    globalroot = html;

    route / {
        autoindex = off;
        index = index.html;
        acceptedmethods = GET;
    }
    
    route /delete {
        autoindex = off;
        index = delete.html;
        root = html;
        acceptedmethods = GET, DELETE;
    }
    
    route /login {
        autoindex = on;
        acceptedmethods = GET;
    }
    
    route /uploads {
        autoindex = on;
        index = upload.html;
        uploaddir = uploads;
        acceptedmethods = GET, POST, DELETE;
    }

    errorpage = 404, html/404.html;
    errorpage = 405, html/405.html;
    errorpage = 500, html/500.html;
]

Configuration Breakdown

port: Defines the port the server listens on.
servernames: Sets the hostname(s) for the server.
bodysizelimit: Maximum size for incoming requests.
globalroot: Root directory for serving files when not specified in route root.
errorpage: Custom error pages for different HTTP error codes.
route: Defines different URL endpoints with settings like index files and accepted methods.
autoindex: Enables or disables directory listing.
acceptedmethods: Specifies allowed HTTP methods.
uploaddir: Defines a directory for file uploads.
root: Root directory for serving files.
